# AI 智能旅游规划项目评估与优化说明

## 一、项目评估发现的问题

### 1. 前端问题

- API 地址硬编码为 `http://localhost:3101` / `http://127.0.0.1:3101`，部署到其他环境后不可复用。
- Vite 配置在 ESM 项目中使用 `__dirname`，在标准 ESM 环境下存在构建失败风险。
- 流式响应解析没有处理 SSE 分包、`complete` / `done` 事件和 HTTP 错误状态，容易出现结束回调不触发或重复触发。
- 首页只能选择城市、预算、天数，缺少旅行偏好、同行人群等个性化参数。
- 详情页没有保存历史行程，用户生成后无法回看。
- “我的”页面功能多为“开发中”，业务闭环不足。
- Chat 页面进入热门城市或详情页咨询时，只填充问题但不自动发送，体验不完整。

### 2. 服务端问题

- 默认端口与前端请求端口不一致，服务端默认为 `3300`，前端请求为 `3101`。
- CORS 手写实现不完整，未覆盖常用请求头，维护成本较高。
- 参数校验散落在路由中，预算、天数等字段没有统一转换与边界校验。
- SSE 工具的 error 事件格式不规范，`end` 和前端解析逻辑不完全匹配。
- 大模型 API Key 缺失或调用失败时，只返回失败，缺少可演示的兜底规划能力。
- Prompt 输出字段较少，缺少本地美食、出行清单、偏好适配等扩展业务信息。

## 二、已完成优化

### 1. 前端修复与重构

- 新增 `src/constants/travel.js`，统一维护城市、旅行偏好、同行人群配置。
- 重写 `src/utils/request.js`，支持环境变量 API 地址、Axios 统一错误处理和健壮 SSE 解析。
- 重写 `Home.vue`，增加旅行偏好、出行人群、历史行程入口和最近规划展示。
- 重写 `Detail.vue`，支持预算明细、本地美食、出行清单、提示事项、兜底规划提示和历史保存。
- 新增 `History.vue`，支持历史行程查看、删除、清空和详情回看。
- 优化 `Chat.vue`，支持场景化自动提问、流式中断、滚动置底和错误提示。
- 优化 `Profile.vue`，接入历史统计、历史清空、重新规划和 AI 咨询入口。
- 修复 `App.vue` 和路由，新增“行程”底部导航。
- 修复 `vite.config.js`，使用 `import.meta.url` 处理别名，并增加 `/api` 代理。

### 2. 服务端修复与重构

- 新增 `src/app.js`，将 Express 应用和监听入口解耦，便于测试和维护。
- 默认端口统一改为 `3101`。
- 使用 `cors` 中间件替代手写 CORS。
- 新增 `src/utils/validators.js`，统一校验行程规划和聊天参数。
- 重写 `src/utils/streamUtils.js`，规范 SSE 的 `message`、`complete`、`done`、`error` 事件。
- 重写 `travelServices.js`，支持旅行偏好、同行人群、本地美食、出行清单等结构化字段。
- 增加本地兜底规划能力：当 API Key 缺失或模型失败时，仍能生成可展示的行程方案。
- 新增 `GET /api/travel/meta`，返回旅行偏好与同行人群枚举。

## 三、扩展后的业务闭环

- 首页填写：城市 + 预算 + 天数 + 旅行偏好 + 出行人群。
- 服务端生成：结构化行程 + 预算拆分 + 本地美食 + 出行清单 + tips + warnings。
- 详情页展示：可查看每日上午、下午、晚上的详细安排。
- 历史行程：自动保存本地历史，支持回看、删除和清空。
- AI 咨询：可基于热门城市或已生成行程继续追问。
- 我的页面：展示历史统计，并提供重新规划、历史记录和 AI 咨询入口。

## 四、验证说明

- 服务端已增加 `npm run check`，用于检查关键 JS 文件语法。
- 前端已增加 `npm run check`，用于检查 Vue SFC 语法。
- 当前环境中 Vite 完整构建受 `node_modules` 缺少 `@rolldown/binding-linux-x64-gnu` 和 `.bin/vite` 无执行权限影响，需要重新安装依赖后再执行完整构建。
